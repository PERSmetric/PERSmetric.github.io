A New Metric for the Evaluation of Fault Localization

Home Archives
2018-01-30

Experimental materials and results of PERS metric

1.Introduction

Many automated fault localization (AFL) techniques have been extensively inverstigated to reduce the developers workload in debugging. One popular and promising catogory of AFL techniques is Spectrum-based Fault Localization (referred to as SFL), which generally produces a rank list of suspicious statements. In the last few years, SFL techniques have been introduced into the fault localization of automated program repair for localizing faults. To evaluate the effectiveness of various SFL techniques, many evaluation metrics, including the commonly used metric EXAM, have been proporsed. However, these metrics have one common limitation that they only focus on the rank of the faulty statement but ignore the accuracy of suspiciousness values, which may cause unfair comparison among SFL techniques. Therefore, they loss part of the information provided by localization technqiues. Take two SFL techniques A and B for example, for the same buggy program, the rank list produced by A is:

RankListA = {(4, 0.9), (7, 0.80), (3, 0.70), (1, 0.6), … }


The rank list produced by B is:

RankListB = {(4, 0.8), (7, 0.75), (3, 0.73), (1, 0.1), … }


where each ordered repair in the rank list records the information of a suspicious statement. The former of the ordered pair indicates the location ID of the statement, and the latter is the relevant suspiciousness.
Judged by EXAM, A and B reach the same performance in localizing this buggy program, but it is obvious that B has shown better accuracy of suspiciousness values, as it increases the suspiciousness of the faulty statement while decreasing that of non-faulty statements compared to A. Therefore, EXAM fails to describe the different performance between A and B as it discards accurate suspiciousness of statements.

Furthermore, these metrics may have a negative impact on the development of SFL techniques. Recall the original definition of suspiciousness: the risk/likelihood of being faulty for a statement. However, Many researchers developed fault localization techniques just for rank and ignored the accuracy of suspiciousness, which deviates from the definition of suspiciousness. For example, Wong3 and Op2 produce negative suspiciousness for some statements, and DStar outputs ‘infinite’ values when scoring statements in some buggy programs.

To address the limitations of existing evaluation metrics, we propose a new metric PERS for evaluating and comparing various SFL techniques, and PERS is particularly suitable to evaluate SFL in the context of automated program repair. We perform experiments on 6 SFL techniques and 22 real-world buggy programs from Defects4J by using the novel automated prorgam repair tool Nopol to verify the feasibility and effectiveness of PERS. Our experimental results suggest that PERS has higher discrimination ability than EXAM, and the SFL technique with higher PERS has more signifcant assisting aids in automated program repair.

2.Experimental Design

There are three metrics involved in the experiments: EXAM, NCP, PERS.
1)EXAM is the percentage of statements to be examined until the faulty statement is found. It is commonly used and is a representative of existing evaluation metrics which only focus on the rank of the faulty statement.
2)NCP is the number of generated patches before a valid patch is found in the repair process. It is an effective measurement to evaluate SFL techniques in automated program repair, but it is limited by the time cost of automated repair on buggy programs, as at least 100 independent automated repair runs are required for each buggy program which may indicate high time cost.
3)PERS is the percentage of suspiciousness value of the faulty statement in the rank list. It is the new metric proposed in this paper and focuses on the accurate suspiciousness of the faulty statement.

We selected Nopol, a novel repair tool for repairing conditional bugs in Java programs, as our experimental tools. Then, we implemented SFA-based Nopol by modifying the RFA-based Nopol (The importance of SFA for automated program repair techniques is demonstrated in our paper). Considering the time cost of automated program repair, we chose 22 real-life bugs, for which valid patches can be generated when repaired by Nopol, from 103 bugs of Defects4J. With respect to SFL techniques, we selected Barinel, Ochiai, Op2, Tarantula and DStar as experimental SFL, because these 5 techniques are popular and well studied in prior studies using EXAM metric. Also, we include Jaccard as it reaches the best performance in the previous study using NCP measurement. As the experiment was conducted in the context of automated program repair, we used the statement, which was modified to generate plausible patch in the repairing process, as the faulty statement.

To verify whether PERS has better discrimination ability than EXAM, we used SFA-based Nopol versions which implemented 6 SFL respectively to repair 22 bugs once to collect the results of PERS and EXAM. In addition, for the purpose of verifying if the automated repair tool with SFL technique having higher PERS has better effectiveness, we performed 100 independent repairing runs for each buggy program and SFL technique to collect the results of NCP based on SFA-based Nopol.

3.Results

Based on the result data and relevant analysis, we have two findings:
1) Compared to EXAM, PERS reaches better performance in distinguishing different SFL techniques.
2) The automated repair tool implementing SFL technique with higher PERS reaches lower NCP in 100 independent repairing trials, and thus achieves better effectiveness in automated repair.

Obviously, PERS has shown more advantages over EXAM, especially in the context of automated program repair. Based on these promising findings, PERS can be used to instruct the development of fault localization techniques to focus more on the accuracy of the suspiciousness value of the faulty statement, which can also benefit automated program repair.
